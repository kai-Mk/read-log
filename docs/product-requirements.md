# プロダクト要件定義書

## 1. プロダクト概要

### 1.1 プロダクト名

read-log

### 1.2 概要

積読本・読みたい本・読了本を管理し、読書状況を可視化するWebアプリケーション。本の状態に応じたユニークなUIで読書体験を楽しくする。

### 1.3 背景・目的

技術書、学術書、小説など多くの本を購入するが、読む前に次の本を買ってしまい積読が増えていく。どの本を持っているか把握しきれず、同じ本を買ってしまうこともある。本の管理と読書状況の可視化によってこの問題を解決する。

また、Claude Codeを使ったスペック駆動開発を体験するための題材として、テスト駆動開発を実践しながら開発する。

## 2. ターゲットユーザー

技術書や学術書を多く購入するが、積読が増えがちなエンジニアや研究者。本の管理に専用ツールは使っておらず、購入した本を把握しきれていない。

ログイン不要で手軽に使いたいユーザー。

## 3. コア機能

### F-01: マイ書庫の作成

**ユーザーストーリー**: 新規ユーザーとして、本を管理する場所を作るために、マイ書庫を作成する機能が欲しい

**受け入れ条件**:

- [ ] 「マイ書庫を作成」ボタンをクリックするとマイ書庫が作成される
- [ ] マイ書庫には一意のIDが付与される
- [ ] 作成後、マイ書庫のURLが表示される
- [ ] URLをブックマークすることで次回以降アクセスできる

### F-02: 本の登録

**ユーザーストーリー**: 積読を管理したいユーザーとして、購入した本を記録するために、本の情報を登録する機能が欲しい

**受け入れ条件**:

- [ ] ISBNを入力すると書籍情報が自動取得される（Google Books API）
- [ ] タイトル、著者、表紙画像、ページ数が表示される
- [ ] ISBN検索で見つからない場合は手動入力できる
- [ ] 登録時に本の状態（積読/読みたい/読了）を選択できる
- [ ] 登録した本が一覧に追加される

### F-03: 本の状態管理

**ユーザーストーリー**: 読書の進捗を管理したいユーザーとして、本の状態を変更するために、ステータス変更機能が欲しい

**受け入れ条件**:

- [ ] 積読・読みたい・読了の3つの状態を切り替えられる
- [ ] 状態変更すると該当するビューに本が移動する

### F-04: 本の編集・削除

**ユーザーストーリー**: 本の情報を修正したいユーザーとして、誤った情報を直すために、本の情報を編集・削除する機能が欲しい

**受け入れ条件**:

- [ ] タイトル、著者、表紙画像、ページ数を編集できる
- [ ] カテゴリ/タグを編集できる
- [ ] 本を削除できる
- [ ] 削除時は確認ダイアログが表示される

### F-05: カテゴリ/タグ管理

**ユーザーストーリー**: 本をジャンル別に整理したいユーザーとして、本を分類するために、カテゴリ/タグを設定する機能が欲しい

**受け入れ条件**:

- [ ] 技術書・小説・学術書・その他のカテゴリを設定できる
- [ ] ISBN検索時にカテゴリが取得できれば自動設定される
- [ ] 自動設定されなかった場合は手動で選択できる
- [ ] カテゴリは後から変更できる

### F-06: メモ機能

**ユーザーストーリー**: 読んだ本の内容を記録したいユーザーとして、感想や要点をメモするために、本にメモを紐付ける機能が欲しい

**受け入れ条件**:

- [ ] 各本にメモを追加できる
- [ ] メモは複数登録できる
- [ ] メモは編集・削除できる
- [ ] メモには作成日時が表示される

### F-07: ユニークUI表示

**ユーザーストーリー**: 楽しく本を管理したいユーザーとして、視覚的に状態を把握するために、状態に応じたユニークなUIで表示する機能が欲しい

**受け入れ条件**:

- [ ] 積読の本は本が積み重なっている状態で表示される
- [ ] 読了の本は本棚に整理されている状態で表示される
- [ ] 読みたい本は本屋さんに並んでいる感じで表示される
- [ ] 各ビュー間でスムーズに切り替えられる

### F-08: 本の一覧・検索

**ユーザーストーリー**: 登録した本を探したいユーザーとして、目的の本を見つけるために、一覧表示と検索機能が欲しい

**受け入れ条件**:

- [ ] 登録した本を一覧で確認できる
- [ ] タイトル、著者で検索できる
- [ ] カテゴリでフィルタリングできる
- [ ] 状態（積読/読みたい/読了）でフィルタリングできる

## 4. 非機能要件

### 4.1 パフォーマンス

- ページ読み込みは3秒以内に完了する
- ISBN検索のレスポンスは2秒以内とする
- 同時接続数は10ユーザーを想定する
- 1ユーザーあたり最大500冊の本を管理できる

### 4.2 セキュリティ

- 認証機能は実装しない（スコープ外）
- マイ書庫IDはURLに含まれ、IDを知っている人のみアクセス可能とする
- マイ書庫IDは推測困難なランダム文字列とする（UUID等）

### 4.3 その他

- Chrome、Firefox、Safariの最新版に対応する
- レスポンシブデザインでモバイルでも利用可能とする
- 日本語のみ対応する

## 5. スコープ外

- ユーザー認証・認可機能
- 複数ユーザー間での本の共有機能
- 読書時間のトラッキング
- 読書目標の設定機能
- SNS連携・シェア機能
- ネイティブモバイルアプリ
- 多言語対応
- 本のデータのインポート/エクスポート機能
