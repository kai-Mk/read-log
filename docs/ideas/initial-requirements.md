# 積読管理と読んだ本の内容を蓄積していくアプリのアイデア

## 背景

エンジニアとして読むべき技術書、趣味で読んでいる学術書、小説などがいろいろありそれを管理しておきたい。また次に読みたい本も整理しておきたいから
またclaude codeを使ったスペック駆動開発を体験するために簡易的なアプリの作成を行う。

## やりたいこと

- 買ったけど読んでいない本、読みたい本、すでに読んだ本をそれぞれ管理する。CRUDの操作
- claude codeの検証なので、最短でできるようにするためにログインなしでも管理できるようなアプリにしたい。
  - 例えばマイ書庫みたいな名前を付けて、それにIDを付与して、そのIDをパスとしてそこからデータを取得する
- UIを面白くしたい。例えば、買ったけど読んでいない本は積読の状態本が積み重なっている状態。すでに読んだ本は、本棚に整理されている状態。読みたい本は本屋さんに並んでいる感じにしたい。
- テストについての知識も深めたいためテスト駆動開発を行い自動テストを追加する

## 追加機能アイデア

### ISBN検索機能

- ISBNを入力すると書籍情報を自動取得（Google Books API）
- 取得情報：タイトル、著者、表紙画像、ページ数
- 入力の手間を大幅に削減

### カテゴリ/タグ機能

- 本のジャンル分け：技術書 / 小説 / 学術書 / その他
- APIでカテゴリが取得できれば自動設定
- 取得できない場合は手動選択、後から編集可能

### メモ機能

- 読んだ本の内容整理・感想を記録
- 本ごとにメモを紐付け

## 機能一覧まとめ

| 機能          | 内容                                         |
| ------------- | -------------------------------------------- |
| 本の管理      | 積読・読みたい・読了の3状態でCRUD            |
| ISBN検索      | タイトル、著者、表紙画像、ページ数を自動取得 |
| カテゴリ/タグ | 技術書/小説/学術書など（API+手動）           |
| メモ機能      | 読んだ本の内容整理・感想を記録               |
| ユニークUI    | 積読=積み重なる本、読了=本棚、読みたい=本屋  |
| 認証なし      | マイ書庫ID方式でアクセス                     |

## 技術スタック

| レイヤー       | 技術                           | 選定理由                                             |
| -------------- | ------------------------------ | ---------------------------------------------------- |
| フロントエンド | React + Vite                   | エコシステム豊富、高速ビルド                         |
| バックエンド   | Hono                           | 軽量、TypeScriptネイティブ、TDDしやすい              |
| ORM            | Prisma                         | 型安全、マイグレーション管理、情報豊富               |
| DB             | PostgreSQL                     | リレーショナル、Prismaと相性良、無料ホスティング充実 |
| テスト         | Vitest + React Testing Library | 高速、設定シンプル、TDD向き                          |
