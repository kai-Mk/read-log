# リポジトリ構造定義書 記述ガイド

## 目的

アーキテクチャ設計書で定義した技術構成を、具体的なディレクトリ構造とファイル配置に落とし込む。

## 各セクションの記述ガイド

### 1. ディレクトリ構造

モノレポ全体のツリー構造を記述する。

- ルートディレクトリから主要なフォルダを階層的に示す
- 各フォルダの用途をコメントで簡潔に記述する
- 深すぎる階層は省略し、重要な部分に焦点を当てる

### 2. 各ディレクトリの責務

主要なディレクトリごとに以下を記述する:

- **パス**: ディレクトリのパス
- **責務**: そのディレクトリが担う役割
- **含まれるファイル**: 配置されるファイルの種類

### 3. レイヤー間データフロー

フロントエンドとバックエンドそれぞれのレイヤー構造とデータの流れを記述する。

- Mermaid記法でフローを図示する
- 各レイヤーがどのディレクトリに対応するかを明記する
- データの流れる方向を矢印で示す

### 4. ファイル命名規則

ファイルの種類ごとに命名規則を定義する。

- **コンポーネント**: PascalCase（例: BookList.tsx）
- **hooks**: camelCaseでuseプレフィックス（例: useBooks.ts）
- **テスト**: 対象ファイル名.test.ts（例: BookList.test.tsx）
- **型定義**: 用途に応じた命名（例: types.ts, book.types.ts）

### 5. モジュール間の依存関係

importのルールと制約を定義する。

- 許可される依存の方向を明記する
- 循環参照を防ぐためのルールを定義する
- 共有モジュールの扱いを記述する
