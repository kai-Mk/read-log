# steering

機能開発のための設計書とタスクリストを作成するスキル。

## 使用可能なツール

- **Read** - ドキュメントを読む
- **Write** - 設計書・タスクリストを作成する
- **Glob** - ファイルを探す
- **AskUserQuestion** - ユーザーに確認する

上記以外のツール（Bash、Edit、Grep、WebFetch、WebSearchなど）は使用しない。

## 目的

機能開発を開始する前に、詳細な設計書とTDDベースのタスクリストを作成する。これにより、実装の方向性を明確にし、効率的な開発を実現する。

## 作成の流れ

1. 既存のドキュメント（PRD、機能設計書、アーキテクチャ設計書、リポジトリ構造）を読み込む
2. 該当する機能の要件と仕様を特定する
3. 機能設計書（design.md）を作成する
4. タスクリスト（tasklist.md）を作成する
5. ユーザーに確認を求める

## 設計書の記述ガイド

### design.md

機能の詳細設計を記述する。以下のセクションを含める:

1. **機能概要**: 機能の目的と概要を1-2文で説明
2. **関連ドキュメント**: PRD/機能設計書の該当セクションへの参照
3. **実装対象**: 作成するファイルの一覧（コンポーネント、hooks、services等）
4. **データフロー**: ユーザーアクションからAPIレスポンスまでの流れ
5. **依存関係**: 使用する共通コンポーネント、外部ライブラリ
6. **注意点**: 実装時に気をつけるべきポイント

### tasklist.md

TDDサイクルに基づいたタスクリストを作成する。以下の形式で記述:

1. **タスクの粒度**: 1タスク = 1テストケース + 1実装
2. **順序**: 依存関係を考慮した順序で記述
3. **チェックボックス**: 進捗を追跡できるようにする
4. **Red/Green/Refactor**: TDDのフェーズを明記

## 出力フォーマット

- `./design.md` - 設計書のテンプレート
- `./tasklist.md` - タスクリストのテンプレート
