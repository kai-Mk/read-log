# Discussion: プロジェクト初期セットアップ

## 2026-01-25

### Q: パッケージバージョンの`^`はパッチバージョンを適用するという意味か？

**回答: 逆。`^`はマイナー・パッチバージョンの更新を許容する。**

| 記法     | 意味                   | 例（`5.3.0`の場合） |
| -------- | ---------------------- | ------------------- |
| `^5.3.0` | マイナー・パッチ更新OK | `5.3.0` ~ `5.x.x`   |
| `~5.3.0` | パッチ更新のみOK       | `5.3.0` ~ `5.3.x`   |
| `5.3.0`  | 完全固定               | `5.3.0`のみ         |

**決定: `^`ありを採用**

- セキュリティパッチが自動適用される
- `pnpm-lock.yaml`で実際のバージョンは固定される

### Q: pnpm-workspace.yamlとルートpackage.jsonは何のために作成するか？

**pnpm-workspace.yaml**

- pnpmにモノレポ構成を認識させる
- `packages/*`配下を1つのリポジトリで管理
- パッケージ間で`workspace:*`を使って相互参照できる（例: frontendからsharedを参照）

**ルートpackage.json**

- ワークスペース全体のスクリプト定義（`pnpm dev`で全体起動など）
- 共通devDependenciesの定義（ESLint, Prettier, TypeScriptを共有）
- 重複インストールを回避
